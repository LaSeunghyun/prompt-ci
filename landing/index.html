Everything you need for prompt quality

prompt-ci is a focused tool. No dashboard signup, no cloud dependency, no vendor lock-in.

.prompt.yaml format

One file per prompt. YAML frontmatter for model settings, Handlebars-style {{variable}} template, and inline test cases — no separate test config needed.

12 assertion types

contains, regex, json-schema, llm-judge, similarity, max-tokens, cost-limit, max-latency, and more. Weight assertions for a composite score.

Multi-provider

OpenAI, Anthropic, and Ollama (local) ship out of the box. Implement the ProviderAdapter interface to add any other provider in ~20 lines.

5 reporters

Console (ANSI colours), JSON, HTML dark theme, Markdown, and CI — which emits GitHub Actions annotations so failures appear inline on your PR diff.

SQLite history

Every run is persisted locally. Query past results, compare scores across commits, track cost trends, and diff prompt versions — all without a cloud service.

Security guardrails

ReDoS protection on all regex assertions, SSRF blocking on provider baseUrl, and path traversal guards on file variable sources. Safe to run in CI by default.

The .prompt.yaml format

Model config, prompt template, variables, and test cases live in one file. No separate fixture files or test runners to wire up.

sentiment-classifier.prompt.yaml yaml 
# Identity
name: sentiment-classifier
model: gpt-4o-mini
temperature: 0

# System prompt
system: |
  You are a sentiment classifier.
  Respond with ONLY one word:
  "positive", "negative", or "neutral".

# Handlebars-style template
template: |
  Classify the sentiment:
  "{{text}}"

# Variable declarations
variables:
  text:
    required: true

# Inline test cases
tests:
  - name: positive-review
    vars:
      text: "I love this product!"
    assertions:
      - type: contains
        value: positive
      - type: max-tokens
        value: 10
      - type: cost-limit
        value: 0.001

  - name: single-word-enforced
    vars:
      text: "It was okay."
    assertions:
      - type: regex
        value: "^(positive|negative|neutral)$"
      - type: llm-judge
        criteria: Response must be exactly one word
promptci.config.yaml yaml 
defaultProvider: openai
defaultModel: gpt-4o-mini

providers:
  openai:
    apiKeyEnv: OPENAI_API_KEY
  anthropic:
    apiKeyEnv: ANTHROPIC_API_KEY
  ollama:
    baseUrl: http://localhost:11434

judge:
  provider: openai
  model: gpt-4o

concurrency: 5
timeoutMs: 30000

reporters:
  - console
GitHub Actions yaml 
name: Prompt CI
on:
  push:
    paths: ['prompts/**']
  pull_request:
    paths: ['prompts/**']

jobs:
  eval:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm install -g prompt-ci
      - name: Run prompt evals
        run: promptci run --reporter ci
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}

How it works

Three steps from writing a prompt to shipping with confidence.

1

Write

Define your prompt template, variables, and test cases in a single .prompt.yaml file alongside your code.

2

Test

Run promptci run. The engine calls your LLM provider, evaluates all assertions, and stores results in SQLite.

3

Ship

Add promptci run --reporter ci to your GitHub Actions workflow. Failed assertions block the merge.
